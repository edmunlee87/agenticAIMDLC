====================================================================
ONE-LINE PURPOSE OF EACH SDK
====================================================================

workflowsdk
- workflow state, routing, checkpoints, resume, recovery, stage control

hitlsdk
- human-in-the-loop review, approval, override, escalation, decision capture

observabilitysdk
- structured event logging, replay, traceability, lineage reconstruction

auditsdk
- formal audit trail, approvals, exceptions, sign-off records

artifactsdk
- artifact registration, metadata, lineage, manifests, lookup

flowvizsdk
- convert logs and workflow actions into flow/timeline visualization

policysdk
- policy packs, threshold rules, breach detection, approval rules

widgetsdk
- reusable Jupyter widget shells for reviews, selection, flow, validation

config_sdk
- config loading, schema validation, versioning, overrides

registry_sdk
- lookup / registry for projects, runs, skills, configs, policies, models

dataset_sdk
- dataset snapshots, split references, lineage, dataset contracts

knowledge_sdk
- governed knowledge objects, promotion workflow, knowledge lifecycle

rag_sdk
- chunking, embeddings, retrieval, reranking, prompt context packaging

dq_sdk
- data profiling and preparation-stage / modeling-stage data quality checks

feature_sdk
- feature engineering, transformations, lagging, metadata, lineage

evaluation_sdk
- metrics, tests, diagnostics, comparisons, threshold evaluation

reporting_sdk
- technical reports, committee packs, summaries, narrative blocks

dataprepsdk
- Spark-first governed training-data preparation using standard templates

monitoringsdk
- post-validation monitoring snapshots, metrics, breaches, dashboard payloads

scorecardsdk
- scorecard-specific logic: binning, WoE, IV, scaling, score bands

timeseriessdk
- time-series logic: stationarity, lags, residual diagnostics, forecasting

eclsdk
- ECL workflow logic: staging, MEV, scenarios, overlays, ECL outputs

lgdsdk
- LGD logic: cure, severity, downturn and forward-looking adjustment

pdsdk
- PD logic: score/rating PD, calibration, term structure, monitoring

eadsdk
- EAD logic: exposure estimation, utilization, CCF-related preparation

sicr_sdk
- SICR logic: threshold comparison, migration, rule/model-based SICR

stresssdk
- stress testing logic: scenario application, stressed outputs, aggregation

validationsdk
- validation workflow, findings, model fitness, conclusions, remediation

agent_bridge
- bounded adapter between agent/skills and SDK calls

api_bridge
- expose SDK capabilities via API

cli_bridge
- expose SDK capabilities via CLI

jupyter_bridge
- connect JupyterLab widgets/workspaces/controllers to SDKs

mcp_bridge
- future MCP-compatible tool exposure layer
