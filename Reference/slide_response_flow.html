Phase 4 — Agent Reasoning Layer (Skill + MCP + SDK Triad)
Every material agent response is composed of:
Skills (Reasoning)
PD review checklist · Candidate comparison guide · Threshold trade-off framework · Methodology note drafting style
Reusable reasoning pattern
MCP Tools (Authoritative)
Fetch candidate metrics · Retrieve approved knowledge · Get validation pack · Fetch run memory · Policy note retrieval
Authoritative system read
SDKs (Execute)
All material actions executed via deterministic, testable SDK functions — not by LLM free-form generation
Deterministic execution
Governance Checkpoint 1 — Candidate Shortlist Review
⬥ Governance Gate: Shortlist Approval
Workflow pauses. HITL review required. Reviewer evaluates candidate metric pack, shortlist recommendation, and trade-off narrative.
Approval Required
→
HITL: Reviewer Decision
Approve / Reject / Return for Rework. Decision, rationale, and reviewer identity durably recorded.
Human-in-the-Loop
→
MCP: Record Approval & Resume
Approval decision committed via governance_sdk. Workflow resumes from checkpoint. Exception path routes back to modelling if rejected.
governance_sdk · workflow_sdk
→
Workflow State: APPROVED → VALIDATION
Shortlisted candidate frozen in registry. Stage transition recorded with full audit trail.
